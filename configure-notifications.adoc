= NetApp Workload Factory 알림 구성
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
NetApp Workload Factory 알림 서비스를 구성하여 NetApp Console의 경고, 계정과 연결된 이메일 주소 또는 Amazon SNS 토픽으로 알림을 보낼 수 있습니다. 경고로 전송된 알림은 Console 에이전트 또는 링크가 배포된 경우 NetApp Console에 표시됩니다. Workload Factory가 Amazon SNS 토픽으로 알림을 게시하면 해당 토픽 구독자(예: 담당자 또는 다른 애플리케이션)는 토픽에 대해 구성된 엔드포인트(예: 이메일 또는 SMS 메시지)로 알림을 수신합니다.

== Workload Factory 알림 구성

NetApp Console 또는 Workload Factory 콘솔을 사용하여 Workload Factory 알림을 구성할 수 있습니다.

알림 수신 옵션은 다음과 같습니다.



=== NetApp 콘솔

* *알림*: NetApp Console의 종 모양 아이콘 아래에 알림이 표시되도록 Workload Factory를 구성하십시오.
* *이메일*: NetApp Console에서 이메일 알림을 활성화하여 Amazon SNS 구성 없이 계정과 연결된 모든 이메일 주소로 알림을 받을 수 있습니다.
* *Amazon SNS*: Workload Factory가 Amazon SNS 주제로 알림을 보내도록 구성합니다.




=== 워크로드 팩토리 콘솔

* *이메일*: Workload Factory 콘솔에서 이메일 알림을 활성화하여 Amazon SNS 구성 없이 계정과 연결된 모든 이메일 주소로 알림을 받을 수 있습니다.
* *Amazon SNS*: Workload Factory 콘솔을 사용하여 Amazon SNS 토픽으로 알림을 보내도록 Workload Factory를 구성합니다.



IMPORTANT: NetApp Console에서 알림을 구성하면, 알림 전달에 대한 소유권이 NetApp Console로 이전되며, Workload Factory 콘솔의 알림 옵션은 비활성화되어 읽기 전용 모드로 표시됩니다. 툴팁에는 알림이 NetApp Console에서 관리된다고 안내되어 있습니다. 이후에는 NetApp Console에서만 알림을 관리할 수 있습니다.

.시작하기 전에
다음 필수 조건은 Amazon SNS 알림을 구성할 때만 적용됩니다.

* Amazon SNS 콘솔이나 AWS CLI를 사용하여 Amazon SNS를 구성하고 Amazon SNS 주제를 생성해야 합니다.
* Workload Factory는 *표준* 주제 유형을 지원합니다.  이 유형의 주제에서는 알림이 수신 순서대로 구독자에게 전송되지 않으므로 중요하거나 긴급한 알림이 있는 경우 이 점을 고려하세요.


[role="tabbed-block"]
====
.NetApp 콘솔에서 알림 구성
--
.단계
. 에 로그인하세요link:https://console.netapp.com["NetApp 콘솔"^] .
. NetApp 콘솔 메뉴에서 *워크로드*, *관리*, *알림 설정*을 차례로 선택합니다.
. 알림 설정 페이지에서 다음 알림 유형 중 하나 이상을 활성화합니다.
+
.. *NetApp Console 알림*: Workload Factory가 NetApp Console에 알림을 보내도록 구성하려면 *NetApp Console 알림 활성화*를 선택하십시오.
.. *이메일 알림*: 계정에 연결된 모든 이메일 주소로 Workload Factory에서 알림을 보내도록 구성하려면 *이메일 알림 활성화*를 선택하십시오.
.. *Amazon SNS 알림*: *SNS 알림 활성화*를 선택한 다음 Amazon SNS 콘솔에서 Amazon SNS를 구성하는 지침을 따르세요.
+
주제를 만든 후 주제 ARN을 복사하여 *알림 설정* 페이지의 *SNS 주제 ARN* 필드에 입력합니다.



. 테스트 알림을 보내 구성을 확인한 후 *적용*을 선택합니다.


--
.Workload Factory 콘솔에서 알림 구성
--
.단계
. 에 로그인하세요link:https://console.workloads.netapp.com["워크로드 팩토리 콘솔"^] .
. Workload Factory 콘솔 메뉴에서 *워크로드*, *관리*, *알림 설정*을 차례로 선택합니다.
. 다음 알림 유형 중 하나 이상을 활성화합니다.
+
** *이메일 알림*: *이메일 알림 활성화*를 선택합니다. 계정과 연결된 모든 이메일 주소가 읽기 전용 모드로 표시됩니다.
** *Amazon SNS 알림*: *SNS 알림 활성화*를 선택한 다음 Amazon SNS 콘솔에서 Amazon SNS를 구성하는 지침을 따르세요.
+
테스트 알림을 보내 구성을 확인한 후 *적용*을 선택합니다.





--

== Amazon SNS 주제 구독하기

Workload Factory를 구성하여 주제에 알림을 보낸 후 다음을 따르세요. https://docs.aws.amazon.com/sns/latest/dg/sns-create-subscribe-endpoint-to-topic.html["지침"] Workload Factory에서 알림을 받을 수 있도록 Amazon SNS 설명서에서 주제를 구독하세요.



== 알림을 필터링합니다

알림에 필터를 적용하면 불필요한 알림 트래픽을 줄이고 특정 사용자에게 특정 유형의 알림만 보낼 수 있습니다. 이 기능은 SNS 알림의 경우 Amazon SNS 정책을 사용하고, NetApp Console의 알림 설정을 통해 구현할 수 있습니다.



=== Amazon SNS 알림 필터링

Amazon SNS 주제를 구독하면 기본적으로 해당 주제에 게시된 모든 알림을 받게 됩니다.  해당 주제의 특정 알림만 받으려면 필터 정책을 사용하여 받을 알림을 제어할 수 있습니다.  필터 정책을 사용하면 Amazon SNS가 필터 정책과 일치하는 알림만 구독자에게 전달합니다.

다음 기준으로 Amazon SNS 알림을 필터링할 수 있습니다.

[cols="3*"]
|===
| 설명 | 필터 정책 필드 이름 | 가능한 값 


| 리소스 유형 | `resourceType`  a| 
* `DB`
* `Microsoft SQL Server host`
* `PostgreSQL Server host`




| 워크로드 | `workload` | `WLMDB` 


| 우선 사항 | `priority`  a| 
* `Success`
* `Info`
* `Recommendation`
* `Warning`
* `Error`
* `Critical`




| 알림 유형입니다 | `notificationType`  a| 
* `Deployment`
* `Well-architected`


|===
.단계
. Amazon SNS 콘솔에서 SNS 주제에 대한 구독 세부 정보를 편집합니다.
. *구독 필터 정책* 영역에서 *메시지 속성*으로 필터링하도록 선택합니다.
. *구독 필터 정책* 옵션을 활성화합니다.
. *JSON 편집기* 상자에 JSON 필터 정책을 입력합니다.
+
예를 들어, 다음 JSON 필터 정책은 WLMDB 작업 부하와 관련된 Microsoft SQL Server 리소스의 알림을 수락하고, 성공 또는 오류의 우선순위를 가지며, Well-architected 상태에 대한 세부 정보를 제공합니다.

+
[source, json]
----
{
  "accountId": [
    "account-a"
  ],
  "resourceType": [
    "Microsoft SQL Server host"
  ],
  "workload": [
    "WLMDB"
  ],
  "priority": [
    "Success",
    "Error"
  ],
  "notificationType": [
    "Well-architected"
  ]
}
----
. *변경 사항 저장*을 선택하세요.


필터 정책의 다른 예는 다음을 참조하세요. https://docs.aws.amazon.com/sns/latest/dg/example-filter-policies.html["Amazon SNS 필터 정책 예시"^] .

필터 정책 생성에 대한 자세한 내용은 다음을 참조하세요. https://docs.aws.amazon.com/sns/latest/dg/sns-message-filtering.html["아마존 SNS 문서"^] .



=== NetApp Console에서 알림 필터링

NetApp Console 알림 설정을 사용하여 콘솔에서 수신하는 알림을 심각도 수준(예: 중요, 정보 또는 경고)별로 필터링할 수 있습니다.

콘솔에서 알림 필터링에 대한 자세한 내용은 다음을 참조하세요. https://docs.netapp.com/us-en/console-setup-admin/task-monitor-cm-operations.html#filter-notifications["NetApp 콘솔 설명서"^] .
