= 사용자 지정 대시보드 문제 해결
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
이 정보를 활용하여 사용자 지정 대시보드를 만들거나 사용할 때 발생할 수 있는 일반적인 문제를 해결하십시오.



[[dashboard-creation-failed]]
== 대시보드 생성 실패

**문제:** 대시보드를 생성할 때 "Failed to understand request and generate code" 오류 메시지가 표시됩니다.

**원인:** 시스템이 사용자의 요청을 이해하지 못했거나 사용자의 프롬프트에서 시각화를 생성할 수 없었습니다.

**해결 방법:** 다음을 시도해 보세요.

* 모니터링하고 싶은 대상을 더 명확하게 명시하여 요청을 다시 작성해 주세요.
* link:create-custom-dashboard.html["사용자 지정 대시보드 만들기"]에 있는 샘플 프롬프트 중 하나로 시작하세요.
* 복잡한 모니터링 요구 사항을 여러 개의 간단한 대시보드로 분할하세요.
* 추적할 워크로드, 메트릭, 시간 범위 또는 차원을 지정하십시오.




[[dashboard-failed-to-load-data]]
== 대시보드에서 데이터를 불러오는 데 실패했습니다.

**문제:** 대시보드가 생성되었지만, 데이터를 불러오는 동안 오류가 발생합니다.

**원인:** 이는 일반적으로 권한 문제 또는 사용 불가능한 데이터 소스 때문에 발생합니다.

**해결 방법:**

* **권한 확인:** Workload Factory 계정에 대시보드에서 사용하는 워크로드 및 리소스에 대한 읽기 액세스 권한이 있는지 확인하십시오. 권한 요구 사항은 link:learn-custom-dashboards.html#aws-credentials-and-permissions["AWS 자격 증명 및 권한"]에서 검토하십시오.
* **자격 증명 확인:** AWS 자격 증명이 유효하고 만료되지 않았는지 확인하십시오. 자세한 내용은 link:add-credentials.html["AWS 자격 증명을 추가합니다"]을 참조하십시오.




[[dashboard-not-loading-or-loading-slowly]]
== 대시보드가 로드되지 않거나 로드 속도가 느립니다.

**문제:** 대시보드 로딩 속도가 느리거나 일부 위젯이 표시되지 않습니다.

**원인:** 로딩 속도가 느린 이유는 네트워크 문제, 대용량 데이터 또는 일시적인 서비스 지연 때문일 수 있습니다.

**해결 방법:**

* **인터넷 연결 상태를 확인하세요:** 인터넷 연결이 안정적인지 확인하십시오.
* **브라우저를 새로고침하세요:** 페이지를 새로고침하여 대시보드가 제대로 로드되는지 확인해 보세요.
* **브라우저 캐시와 쿠키를 삭제하세요.** 이렇게 하면 오래되었거나 손상된 데이터로 인해 발생하는 로딩 문제를 해결하는 데 도움이 될 수 있습니다.
* **잠시 기다렸다가 다시 시도하세요:** 문제가 계속되면 몇 분 정도 기다렸다가 다시 시도해 보세요.




[[missing-dashboards]]
== 누락된 대시보드

**문제:** 사용자가 생성한 대시보드가 대시보드 카탈로그에 더 이상 표시되지 않습니다.

**원인:** Workload Factory 계정의 다른 사용자가 해당 파일을 삭제했을 수 있습니다.

**해결 방법:**

* 올바른 Workload Factory 계정으로 로그인했는지 확인하십시오.
* 계정에 있는 다른 사용자에게 누군가 대시보드를 삭제했는지 물어보세요.
* 필요한 경우, 원래 요구 사항을 사용하여 대시보드를 다시 생성하십시오.


Workload Factory 계정의 모든 사용자는 사용자 지정 대시보드를 보고 삭제할 수 있습니다. 공유 대시보드를 관리할 때는 팀원들과 협업하십시오.
